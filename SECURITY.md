# Security Policy

## Supported Versions

Only the latest published release (the most recent GitHub Release / F-Droid version) is
supported. There are no parallel maintained release branches.

## Reporting a Vulnerability

Gitling handles sensitive material (SSH keys, git credentials, repository contents), so please
**do not open a public GitHub issue** for a suspected security vulnerability.

Instead, report it privately using one of:

- GitHub's [private vulnerability reporting](../../security/advisories/new) for this repository
  (Security tab → "Report a vulnerability").
- Email: bmaneeshacooray@gmail.com

Please include:
- A description of the vulnerability and its potential impact.
- Steps to reproduce, or a proof of concept if available.
- The app version and Android version/device you tested on.

This is a single-maintainer project — there's no formal SLA, but reports will be acknowledged
and addressed as soon as possible.
